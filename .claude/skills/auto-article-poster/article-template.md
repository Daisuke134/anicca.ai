- [学べること3]

[1-2文で説明]

## セクション別ガイド

### タイトル

| ルール | 例 |
|--------|-----|
| 30文字以内 | 「6年間失敗した習慣化がAIで変わった」 |
| 数字を入れる | 「10個挫折」「Day 15」 |
| 変化を示す | 「〜だった→〜になった」 |

### この記事で学べること

**目的:** 読者が読む価値を判断できるようにする

```markdown
## この記事で学べること
- Claude Codeで自動投稿スキルを作る方法
- note.comへの自動投稿の実装パターン
- 100%自動化ではなく95%自動化で十分な理由
```

### 何を作ったのか？

**目的:** 1-2文で成果物を明確にする

```markdown
## 何を作ったのか？

「noteに投稿して」と言うだけで、SEO対策されたBuild in Public記事が自動生成されるスキルを作った。

**一言で言うと**: 作業したら自動で記事ができる仕組み
```

### なぜ作ろうと思ったのか？

**目的:** ペルソナとの共感を生む

**必須要素:**
1. 苦しみの共有（6年間失敗、10個挫折、自己否定）
2. 転機となった気づき
3. 解決への希望

```markdown
## なぜ作ろうと思ったのか？

Build in Publicを続けたい。でも毎日記事を書く時間がない。

- 開発作業だけで精一杯
- 記事を書くのに30分以上かかる
- SEO対策とか考え始めると終わらない

「作業したら自動で記事ができたらいいのに」

ずっとそう思っていた。Claude Codeのスキル機能を知って、「これだ」と思った。
```

### どうやって作ったのか？

**目的:** 実装の詳細を共有する（再現可能なレベルで）

**必須要素:**
1. ステップバイステップの説明
2. 「ハマったところ」の正直な共有
3. 各ステップでの学び

```markdown
### Step 1: 全体設計を決めた

まず、何を自動化するか整理した。

1. 作業内容から記事のネタを抽出
2. SEO対策されたタイトルを生成
3. テンプレートに沿って本文を生成
4. note.comに自動投稿（下書き保存まで）

**ポイント**: 最初から完璧を目指さず、「下書き保存まで」に絞った。
公開は手動で確認してから。
```

### 学んだこと・気づき

**目的:** 読者が持ち帰れる教訓を提供する

```markdown
## 学んだこと・気づき

1. **完璧を目指すと終わらない**: 100%自動化に3日費やしたが、
   95%で十分だった。「ユーザーにCmd+Vしてもらう」という妥協で解決。

2. **モジュール分割は最初からやるべき**: 後から分割すると手間がかかる。
   最初から役割ごとにファイルを分けておくと、メンテナンスも共有も楽。

3. **自分用ツールでも共有前提で作る**: 将来の自分も「他の人」と同じ。
   READMEを書いておくと、数ヶ月後に「これ何だっけ」と悩まなくて済む。
```

### まとめ

**目的:** 記事の締めくくり + 読者へのアクション促進

```markdown
## まとめ

「作業したら自動で記事ができる」という夢が、Claude Codeのスキル機能で実現できた。

完全自動化は難しくても、95%の自動化で十分価値がある。

同じようなことをやってみたい方の参考になれば嬉しい。
質問があればコメントで教えてください。
```

## 禁止事項

| 禁止 | 理由 | 代替 |
|------|------|------|
| 「簡単に！」 | ペルソナが警戒 | 「ステップバイステップで」 |
| 技術用語の羅列 | 読者が離脱 | 一般用語に変換 |
| 成功だけを書く | 信頼性低下 | 失敗談も正直に |
| 抽象的な表現 | 具体性がない | 数字や具体例を使う |
| 長すぎる段落 | 読みにくい | 3-4文で区切る |
