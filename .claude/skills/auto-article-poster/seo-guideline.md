# SEO Guideline for Anicca Articles

note.com向けの記事でSEOを最適化するためのガイドライン。

---

## キーワード変換表

技術用語を一般読者に伝わる言葉に変換する。

| 技術用語 | 一般用語 | 使用例 |
|---------|---------|--------|
| API連携 | 自動化 | 「APIを叩いて〜」→「自動化して〜」 |
| LLM | AI | 「LLMで生成」→「AIで生成」 |
| プロアクティブ通知 | 自動リマインド | 「プロアクティブに介入」→「自動でリマインド」 |
| 行動変容 | 習慣化サポート | 「行動変容を促す」→「習慣化をサポートする」 |
| Nudge | やさしい後押し | 「Nudgeを送信」→「やさしく後押し」 |
| SwiftUI | iOSアプリ | 「SwiftUIで実装」→「iOSアプリを作った」 |
| プロンプトエンジニアリング | AIへの指示の工夫 | 「プロンプトを調整」→「AIへの指示を工夫」 |
| リファクタリング | コードの整理 | 「リファクタリング」→「コードを整理」 |
| デプロイ | 公開/リリース | 「本番にデプロイ」→「公開」 |
| CI/CD | 自動テスト | 「CIを通す」→「自動テストを走らせる」 |

## タイトルパターン（バズりやすい型）

### パターン1: 数字 + 変化

**効果:** 具体性 + 変化の期待

| パターン | 例 |
|---------|-----|
| ○年間失敗 → 変化 | 「6年間失敗し続けた習慣化が、AIで変わった」 |
| ○個試して失敗 → 成功 | 「習慣アプリ10個挫折した人間が、自分で作った話」 |
| ○日で○○ | 「7日間でAIエージェントを作った記録」 |

### パターン2: 問題 + 解決

**効果:** 共感 + 解決への期待

| パターン | 例 |
|---------|-----|
| 〜だった私が | 「毎日3日坊主だった私が、AIに助けられた話」 |
| 〜できなかった理由 | 「習慣化できなかった本当の理由と、その解決策」 |
| 〜で悩んでいる人へ | 「自分を責め続けている人に伝えたいこと」 |

### パターン3: ストーリー

**効果:** 物語への引き込み

| パターン | 例 |
|---------|-----|
| 〜した話 | 「AIに記事を書かせるようにした話」 |
| 〜の記録 | 「プロアクティブAIエージェントを作る記録 Day 15」 |
| なぜ〜を作ったのか | 「なぜ習慣アプリを自分で作ることにしたのか」 |

### パターン4: ハウツー

**効果:** 実用性 + 検索流入

| パターン | 例 |
|---------|-----|
| 〜する方法 | 「AIに毎日記事を書いてもらう方法」 |
| 〜したら〜になった | 「○○を自動化したら毎日の発信が10倍楽になった」 |
| 〜を作る手順 | 「Claude Codeで自動投稿スキルを作る手順」 |

### パターン5: 意外性

**効果:** 好奇心 + クリック率

| パターン | 例 |
|---------|-----|
| 実は〜 | 「実は習慣化できない人ほどAIと相性がいい」 |
| 〜と思っていたけど | 「習慣アプリは意味ないと思っていたけど」 |
| 〜するな | 「最初から完璧を目指すな」 |

## 禁止パターン（ペルソナが警戒する）

| 避けるべきパターン | 理由 |
|-------------------|------|
| 「簡単に〜！」 | 6年間失敗した人は「簡単」を信じない |
| 「たった○日で！」 | 何度も騙されてきた |
| 「誰でもできる！」 | 「自分にはできない」と思い込んでいる |
| 「奇跡の〜」 | 大げさすぎて信用されない |
| 「これさえあれば〜」 | 銀の弾丸は存在しないと知っている |

## ハッシュタグ戦略

### 必須タグ（毎回使う）

```
#BuildInPublic #個人開発 #AI活用
```

### 内容に応じて追加

| カテゴリ | タグ |
|---------|------|
| 習慣化系 | `#習慣化` `#行動変容` `#自己改善` |
| 開発系 | `#iOSアプリ` `#Swift` `#モバイルアプリ` |
| AI系 | `#ChatGPT` `#Claude` `#AIエージェント` |
| マインド系 | `#自己肯定感` `#メンタルヘルス` `#マインドフルネス` |

### タグの数

- **推奨: 5-7個**
- 多すぎるとスパムに見える
- 少なすぎると発見されにくい

## メタ情報の最適化

### 冒頭100文字（プレビュー表示）

note.comでは冒頭がプレビューに表示される。

**良い例:**
```
6年間、習慣化に失敗し続けてきた。10個以上のアプリを試して、全部3日坊主。
「自分はダメな人間だ」と思い込んでいた。でも今、自分でAIエージェントを作っている。
```

**悪い例:**
```
こんにちは！今日は習慣化について書きます。私は以前から習慣化に興味があって...
```

### サムネイル画像

| 要素 | 推奨 |
|------|------|
| サイズ | 1200x630px（OGP推奨） |
| 文字 | 読みやすい大きさ、白抜きorシャドウ |
| 色 | コントラスト強め、目立つ色 |
| 内容 | タイトルの要約 or キービジュアル |

## 検索キーワード（SEO狙い）

Aniccaの記事で狙うべきキーワード:

| キーワード | 検索意図 | 競合 |
|-----------|---------|------|
| 習慣化 アプリ おすすめ | アプリを探している | 高 |
| 習慣化 できない 理由 | 解決策を探している | 中 |
| 三日坊主 克服 | 失敗から脱出したい | 中 |
| AI 習慣化 | 新しい方法を探している | 低 |
| 行動変容 アプリ | 専門的な解決策を探している | 低 |
