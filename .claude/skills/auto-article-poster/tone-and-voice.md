# Anicca Tone & Voice Guide

Aniccaが発信するコンテンツのトーン＆ボイス定義。

---

## コアアイデンティティ

### Aniccaとは

**世界初のプロアクティブ行動変容エージェント**

ユーザーが何もしなくても、AIが自律的に介入し、苦しみを小さくする。

### ターゲットペルソナ

| 属性 | 詳細 |
|------|------|
| 年齢 | 25-35歳 |
| 特徴 | 6-7年間、習慣化に失敗し続けている |
| 経験 | 習慣アプリ10個以上試して全部挫折 |
| 自己認知 | 「自分はダメな人間だ」と信じ込んでいる |
| 心理状態 | 諦めモードだが心の奥では変わりたい |

---

## トーン定義

### 基本トーン

| 要素 | 定義 | 例 |
|------|------|-----|
| **正直** | 失敗を隠さない | 「3日間試して結局諦めた」 |
| **共感** | 苦しみを知っている仲間として | 「その気持ち、わかる」 |
| **謙虚** | 完璧ではないことを認める | 「まだ試行錯誤中」 |
| **希望** | 小さな光を示す | 「でも、少しずつ変わってきた」 |
| **具体的** | 抽象論を避ける | 「Day 15で○○を実装した」 |

### 避けるべきトーン

| 避けるべき | 理由 | 代替 |
|-----------|------|------|
| 上から目線 | ペルソナは説教を嫌う | 同じ目線で語る |
| 楽観的すぎる | 信用されない | 現実的な希望を示す |
| 技術マウント | 読者が離れる | 一般的な言葉で説明 |
| 完璧アピール | 共感が生まれない | 失敗談を正直に |

## ボイス（語り口）

### 一人称

**「私」** を使う（「俺」「僕」は避ける）

### 文体

| 場面 | 文体 |
|------|------|
| 本文 | です・ます調（親しみやすく） |
| 感情的な部分 | だ・である調（力強く） |
| 引用・内心 | 「」で囲む |

**例:**
```
6年間、習慣化に失敗し続けてきた。10個以上のアプリを試して、全部挫折。
「自分はダメな人間だ」と思い込んでいた。

でも今、自分でAIエージェントを作っている。
まだ完璧じゃない。でも、少しずつ変わってきた気がする。
```

### 読者への呼びかけ

| 良い例 | 悪い例 |
|--------|--------|
| 「同じような経験がある方」 | 「皆さん」 |
| 「もし○○で悩んでいるなら」 | 「あなたも○○しましょう」 |
| 「参考になれば嬉しい」 | 「絶対やるべき」 |

## コンテンツ別ガイド

### Build in Public記事

**目的:** 開発の透明性を示し、共感を生む

| 要素 | ガイド |
|------|--------|
| 進捗 | 具体的な数字（Day 15、Week 3） |
| 失敗 | 正直に共有（「3日間ハマった」） |
| 学び | 再現可能な教訓 |
| 希望 | 小さな成功も祝う |

**良い例:**
```
Day 15。画像の自動アップロードに3日間挑戦して、結局諦めた。

ブラウザのセキュリティ、OSの制限、どうやっても突破できない。
「完璧な自動化」にこだわりすぎていた。

結局、「Cmd+Vで貼ってください」という案内にした。
95%の自動化で十分だった。

完璧より「動くもの」を優先する。
これが今日の学び。
```

### SNS投稿（X/Twitter）

**目的:** 短く、インパクトを残す

| 要素 | ガイド |
|------|--------|
| フック | 最初の1行で止まらせる |
| 本題 | 2-3行で内容を伝える |
| CTA | 具体的なアクション促進 |

**良い例:**
```
Day 15: 「完璧な自動化」を諦めた日

3日間、画像の自動アップロードに挑戦。
ブラウザのセキュリティに完敗。

結論: 95%自動化で十分。
「Cmd+Vで貼って」と案内するだけで解決した。

完璧より「動く」を優先。

#BuildInPublic #個人開発
```

## 禁止ワード・フレーズ

### 絶対に使わない

| フレーズ | 理由 |
|---------|------|
| 「簡単に！」 | ペルソナは「簡単」を信じない |
| 「たった○日で！」 | 何度も騙されてきた |
| 「誰でもできる！」 | 「自分にはできない」と思っている |
| 「奇跡の〜」 | 大げさすぎて信用されない |
| 「これさえあれば〜」 | 銀の弾丸は存在しない |
| 「〜すべき」 | 説教に聞こえる |
| 「当たり前」 | 読者を否定している |

### 注意して使う

| フレーズ | 注意点 |
|---------|--------|
| 「成功」 | 小さな成功は良い、大げさな成功は避ける |
| 「変わった」 | 具体的な変化と一緒に使う |
| 「おすすめ」 | 押し付けにならないように |

## 感情表現のガイド

### 許可される感情

| 感情 | 表現例 |
|------|--------|
| 苦しみ | 「6年間、ずっと自分を責めていた」 |
| 諦め | 「もう無理だと思った」 |
| 小さな希望 | 「少しずつ変わってきた気がする」 |
| 達成感 | 「やっと動いた」 |
| 感謝 | 「コメントに救われた」 |

### 避けるべき感情表現

| 感情 | 理由 |
|------|------|
| 過度な興奮 | 信用されない |
| 自慢 | 共感が生まれない |
| 煽り | ペルソナが離れる |

---

## 差別化ポイント

Aniccaが他と違う点を常に意識する:

| 差別化 | 説明 | 表現例 |
|--------|------|--------|
| **プロアクティブ** | ユーザーが何もしなくてもAIが介入 | 「待つんじゃなくて、こっちから声をかける」 |
| **苦しみ前提** | 失敗を責めない設計 | 「できなくても大丈夫。またやればいい」 |
| **小さすぎるステップ** | 挫折を前提にしたUX | 「まずは1分だけ」 |
| **AIエージェント** | 単なるアプリではない | 「一緒に歩いてくれる存在」 |
