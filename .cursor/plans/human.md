# Anicca iOS 動画撮影対応機能 実装仕様書

## 1. 背景と目的

### 1.1 現状（As Is）

現在のAnicca iOSアプリには以下の機能が実装されている：

- **習慣リスト画面**: Wake/Training/Bedtime/Custom習慣がトグル付きで表示される。トグルは習慣の有効/無効（通知スケジュールの登録/解除）を制御する。
- **習慣セッション画面**: 通知タップで遷移し、Aniccaが音声で話しかける。セッション終了時は×ボタンでHabits画面に戻るのみ。
- **通知機能**: 習慣ごとに設定された時刻に通知が発火し、タップでセッション画面に遷移する。
- **TalkView**: トピック選択からセッションを開始する画面。

しかし、以下の機能が存在しないため、マーケティング動画の撮影ができない：

1. **日次完了チェック機能がない**: ユーザーが今日その習慣を実行したかどうかを記録・表示する仕組みがない。習慣リストには「通知ON/OFF」のトグルしかなく、「今日やったか」を示すUIがない。

2. **セッション完了後の確認フローがない**: セッション終了時に×ボタンを押すと、単にHabits画面に戻るだけ。「起きられたか」「トレーニングを終えたか」を確認する仕組みがなく、成功体験を演出できない。

3. **成功時のフィードバックがない**: 習慣を達成した際に、ユーザーを称えるダイアログやアニメーションがない。達成感を演出できず、動画として見栄えがしない。

4. **ミニ説法機能がない**: 習慣がうまくいかなかった日に戻れる「説法ルーム」のような機能がない。

5. **今日の一言通知がない**: 毎日12時に送られる抽象的なメッセージ通知と、その表示画面がない。

これらの不足により、以下のマーケティング動画を撮影することができない：
- 起床アラーム→セッション→「起きられた」→チェックONの一連の流れ
- 就寝リマインド→セッション→「今寝る」→チェックONの一連の流れ
- トレーニング通知→セッション→「終わった」→チェックONの一連の流れ
- 習慣リスト画面でチェックが入っていく様子
- ミニ説法タブと再生画面
- 今日の一言通知と表示画面

### 1.2 目標とする状態（To Be）

本実装により、以下の状態を実現する：

**A. 日次完了チェック機能**

習慣リスト画面の各行に、既存のトグル（通知ON/OFF）とは別に、「今日の完了チェック」を示すチェックボックスを追加する。未完了は空の四角（☐）、完了済みはチェック入り四角（☑）で表示する。日付が変わると全チェックボックスがリセットされる。手動でのタップによるON/OFF操作も可能とする。

**B. セッション完了確認フロー**

セッション画面で×ボタンを押した際、下からシートが出現し、ユーザーに習慣の完了/未完了を確認する。選択肢は習慣タイプごとに異なる：

- 起床: 「二度寝する」「起きた」
- トレーニング: 「諦める」「終わった」
- 就寝: 「夜更かしを続ける」「今寝る」
- カスタム: 「諦める」「完了した」

完了を選択した場合、Habits画面に遷移後、成功ダイアログが表示される。ダイアログには祝いの絵文字（🎉）と2行のメッセージが表示される。メッセージは10種類用意し、日付ベースでローテーションする。ダイアログのOKボタンを押すと、該当習慣のチェックボックスが自動でONになる。

未完了を選択した場合、Habits画面に遷移するのみで、ダイアログもチェックも入らない。

**C. 全習慣での統一フロー**

就寝のみ特別な中間画面を用意するのではなく、全習慣で同じフロー（通知→セッション→×ボタン→完了確認シート→成功ダイアログ→チェックON）を採用する。これにより、UXの一貫性が保たれ、実装もシンプルになる。

**D. ミニ説法タブとプレイヤー（後続実装）**

Bottom tabに新しいタブ「Dharma」を追加し、説法カードの一覧と再生画面を実装する。ただし、本フェーズでは優先度が低いため、後続実装とする。

**E. 今日の一言通知（後続実装）**

毎日12時に固定スケジュールで抽象メッセージを通知し、タップで簡易表示画面に遷移する機能を実装する。ただし、本フェーズでは優先度が低いため、後続実装とする。

### 確認

元の要件を見ると：

> 通知タップ時に開く簡易画面（同じ一言を大きく表示するだけ）を用意できると動画にも使いやすい。

とあったので「表示画面」と書いた。

ただ、今の質問を見ると「ただ通知が来るだけ」と言っているので、どちらにするか確認させてほしい：

**A案: 通知だけ**
- 毎日12時に通知が来る
- タップしても特に何も開かない（アプリが開くだけ）
- 実装がシンプル

**B案: 通知＋表示画面**
- 毎日12時に通知が来る
- タップすると、その一言が大きく表示される簡易画面が開く
- 動画で「通知タップ→一言表示」の流れを撮影できる

### 6.4 音声ファイルについて

**質問:** 音声ファイルはどうする？

**A案: ローカルバンドル**
- 事前に録音した音声をアプリにバンドル
- オフラインでも再生可能
- アプリサイズが増える

**B案: サーバーから取得**
- サーバーに音声ファイルを配置
- 再生時にストリーミング/ダウンロード
- アプリサイズは増えない
- ネットワーク必須

**C案: ダミー音声で動画撮影**
- とりあえず無音または仮音声で実装
- 動画撮影時は画面だけ撮影
- 本番音声は後で差し替え

どれにする？
