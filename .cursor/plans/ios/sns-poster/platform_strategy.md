## 運用フロー（投稿 → 記録）

### 1. 投稿前チェック

```
1. schedule_log.json を読み込む
2. 今日の日付のスロットを確認
3. 対象アカウントの null（空き）スロットを特定
4. 衝突する時間がないか確認
```

### 2. 投稿実行

```
1. コンテンツ生成（画像/動画/テキスト）
2. Blotato APIでスケジュール投稿
3. 投稿成功を確認
```

### 3. 記録更新（重要！）

投稿後、`schedule_log.json` を更新：

```json
{
  "2026-01-21": {
    "x_xg2grb": {
      "08:00": null,
      "12:30": {
        "type": "BIP",
        "content": "Day 20 of building Anicca",
        "posted_at": "2026-01-21T12:30:00+09:00"
      },
      "21:00": null
    }
  },
  "history": [
    {
      "date": "2026-01-21",
      "time": "12:30",
      "account": "x_xg2grb",
      "type": "BIP",
      "content": "Day 20 of building Anicca"
    }
  ]
}
```

### 4. 同時投稿を避けるルール

- **同一プラットフォーム内**: 最低30分空ける
- **異なるプラットフォーム**: 同時刻OK（むしろ推奨）
- **同一アカウント**: 各スロット時間を厳守
