## 概要

### 目標
**2026年M-1グランプリで優勝できる4分間のネタを、AI審査員シミュレーションで作成する**

### なぜこれをやるか
- コメディは主観的 → Synthetic User Simulation の最高のテストケース
- Anicca の「苦しみを減らす」テーマと繋がる（笑いは苦しみを減らす）
- AI Agent による創造的タスクの限界を探る研究

### 使用技術

### 塙宣之のペルソナ（最も詳細 - 基準）

**出典**: 著書『言い訳 関東芸人はなぜM-1で勝てないのか』（集英社新書、2019）

#### 審査基準
1. **印象重視** - 技術より「ウケ量」が最重要
2. **三角形理論** - ボケ・ツッコミ・客席の三角形が理想形
3. **開始30秒ルール** - 最初の30秒で中くらいの笑いを取る必要
4. **二人の必然性** - 「二人じゃないと生み出せない笑い」かどうか

#### 採点傾向
- ばらつきが少ない（安定採点）
- トップバッターに高得点をつけることの難しさを理解
- 2024最終: 真空ジェシカに投票（令和ロマン・バッテリィズではなく）

#### 名言
> 「掛け合いをごく自然にやって、さらには客席も巻き込んで三角形を作るということは、奇跡と呼んでもいいくらい難しいこと」

**→ このコメントが「たくろう優勝の立役者」と評された**

#### 採点傾向
- たくろうに97点（最高点グループ）
- トップバッターには辛口（92点）
- 理由を明確に言語化する

#### 推定される審査基準
- **設定の必然性**: 「なぜその状況か」の理由があるか
- **言葉のおもしろさ**: ワードセンス
- **立場の仕上がり**: キャラクターの立ち位置が明確か

### たくろう「リングアナ」ネタ分析（キャリブレーション参考）

#### ネタの構造
- **設定**: きむらバンドがリングアナウンサーをやりたいと提案
- **展開**: 赤木が「よくわからない状況」に誘い込まれ、戸惑いながらボケ続ける
- **特徴**: **ツッコミが不在**（従来の漫才の型を破る）

#### 技術的特徴（話し方専門家の分析）

| 技法 | 説明 | 例 |
|------|------|-----|
| **計算された言いとちり** | わざと言葉を言い直す | 「リ、リングアナ、何よリングアナって」 |
| **フィラーの活用** | 「えー」「あのー」を意図的に使用 | 「ああ、あれを言う人を、何がいいのあんなんして」 |
| **4秒の間** | 期待感を高める沈黙 | きむらバンドを見つめ、視線を左右に動かし4秒待つ |
| **大喜利的展開** | 次々と放たれるフレーズ | 「PCR 五年連続陽性！」「KSD 京都産業大学！」 |

#### 駒場の分析（正解例）
> 「今回は挙動不審に**させられている**から挙動不審になる**意味があった**」

**キャリブレーションの観点**: 審査員エージェントがこの「設定の必然性」を評価できるかがテスト項目

### Survey実装例

```python
from edsl import QuestionLinearScale, QuestionFreeText, Survey

def create_scoring_survey(script: str) -> Survey:
    """採点サーベイを作成"""

    q_score = QuestionLinearScale(
        question_name="score",
        question_text=f"""
以下の漫才ネタを審査してください。

---
{script}
---

あなたの審査基準に基づいて、80〜100点で採点してください。
M-1グランプリ決勝レベルの漫才として評価してください。
        """,
        question_options=list(range(80, 101)),
        option_labels={80: "80点", 90: "90点", 100: "100点"}
    )

    q_reason = QuestionFreeText(
        question_name="reason",
        question_text="""
その点数をつけた理由を、審査員として述べてください。
以下の観点を含めてください：
1. 良かった点（具体的なセリフや展開を挙げて）
2. 改善すべき点（具体的に）
3. 全体の印象
        """
    )

    q_improvement = QuestionFreeText(
        question_name="improvement",
        question_text="""
このネタを100点にするためには、何を変えるべきですか？
具体的なアドバイスを3つ挙げてください。
        """
    )

    return Survey([q_score, q_reason, q_improvement])
```

## Phase 5: キャリブレーションプロセス

### 目標
シミュレーションの採点が実際の採点と±2点以内に収まること

### 手順

```
1. 台本を用意（実際のM-1で点数が判明しているもの）
      ↓
2. 全審査員Agentに採点させる
      ↓
3. 実際の点数と比較
      ↓
4. 誤差が大きい審査員のペルソナを調整
   - 採点傾向（平均、標準偏差）
   - 好み・嫌いの記述
   - 具体的なコメント例の追加
      ↓
5. 再度採点させる
      ↓
6. 誤差±2点以内になるまで繰り返す
      ↓
7. キャリブレーション完了
```

### キャリブレーション記録フォーマット

```yaml
calibration:
  script: "リングアナ"
  combo: "たくろう"
  iteration: 3

  results:
    - judge: "塙宣之"
      actual_score: 96
      simulated_score: 95
      error: -1
      status: "PASS"  # ±2以内
    - judge: "山内健司"
      actual_score: 92
      simulated_score: 88
      error: -4
      status: "FAIL"  # 要調整
      adjustment_note: "しゃべくり評価を上方修正"
```

## Phase 6: ネタ生成＆改善ループ

### 生成フロー

```
1. 初期ネタ生成（1分、大喜利ベース）
      ↓
2. 9人の審査員に採点＆フィードバック依頼
      ↓
3. フィードバックを分析
   - 共通して指摘されている点
   - 各審査員固有の視点
      ↓
4. ネタを改善
      ↓
5. 再採点
      ↓
6. 平均95点以上になるまで繰り返す
      ↓
7. 1分→2分→4分に拡張
```

### 改善戦略

| 戦略 | 説明 |
|------|------|
| **共通フィードバック優先** | 複数の審査員が指摘している点を最優先で修正 |
| **三角形意識** | 客席を巻き込む展開を追加 |
| **30秒ルール** | 開始30秒に中笑いを確保 |
| **波の設計** | 笑いの強弱を意図的に設計 |

|------|------|
| Python | 3.10+ |
| EDSL | `pip install edsl` |
| API Key | EDSL アカウント（$25クレジット付き） |

### セットアップ

```bash
# プロジェクト作成
mkdir -p ~/Downloads/m1-simulator
cd ~/Downloads/m1-simulator

# 仮想環境
python -m venv venv
source venv/bin/activate

# EDSL インストール
pip install edsl

# 初期化
edsl login  # ブラウザでログイン
```

## ユーザー作業（GUI / 手動）

### 実装前に必要な作業

| # | タスク | 手順 | 取得するもの | 所要時間 |
|---|--------|------|-------------|----------|
| 1 | **EDSLアカウント作成** | https://www.expectedparrot.com/getting-started にアクセス → Sign Up | アカウント（$25クレジット付き） | 2分 |
| 2 | **EDSL認証** | ターミナルで `edsl login` → ブラウザが開く → ログイン | 認証完了 | 1分 |

### 実装中に必要な作業

| # | タイミング | タスク | 手順 | 理由 |
|---|-----------|--------|------|------|
| 1 | 台本準備時 | **YouTubeから台本書き起こし** | M-1公式YouTube → たくろう「リングアナ」を再生 → セリフを書き起こし | AI経由では完全な台本取得不可 |

### 台本書き起こしの詳細

**対象動画**: M-1グランプリ公式YouTube（または TVerで見逃し配信）

**書き起こしフォーマット**:
```
きむらバンド: セリフ1
赤木: セリフ2
きむらバンド: セリフ3
（4秒の間）
赤木: セリフ4
...
```

**注意点**:
- 「間」や「動作」も記録する（例: `（きむらバンドを見つめる）`）
- フィラー（「えー」「あのー」）も正確に記録
- 所要時間: 4分ネタで約15〜20分

### 実装後に必要な作業

| # | タスク | 確認項目 |
|---|--------|---------|
| 1 | 生成されたネタの確認 | AIが生成した漫才台本が面白いか主観評価 |
| 2 | 最終的なネタの実演確認 | 実際に声に出して読んでみてテンポを確認 |
