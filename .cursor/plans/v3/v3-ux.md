# Anicca UX仕様書（理想体験・完全版）

## 1. プロダクト概要

Anicca は「行動と人格を導く、音声の導師（Persuasion Agent）」。

怠惰で、自分で決められず、自信もなくて、何度も三日坊主で終わってきた人でも
Anicca がその人の生活・苦しみ・理想を理解し、

適切なタイミングで
適切な言葉と声で
強く、優しく、時に厳しく

その人を理想の方向へ引っ張っていく。

ゴールは、
「外の Anicca（アプリ）」が

1つ1つの Nudge を通して
ユーザーの中に「内なる Anicca」の種を植え、育て、

最終的には、
外からの通知がなくても、
その人自身の内側の声が、自然に自分を導けるようになること。

まさに、ブッダが説法によって人々を苦しみから解き放ち、
「犀の角のようにただ独り歩める」状態へ連れていったように、
Anicca はユーザーを「自分の足で歩ける地点」まで送り届けたら、
静かに背景へ退いていく。

この UX 仕様書は、そのための「理想的な体験」の全体像を描く。

Anicca が実現したい 3 つのコア体験は次の通り。

1. 苦しみのど真ん中で、ボタン一つで導師が迎えに来る体験
2. ユーザーの行動ログに基づき、Anicca 自身が「勝てる Nudge」を設計・自律改善していく体験
3. Behaviorタブで「今の自分」と「今の行動が続いた未来」が見え、
   さらに月日とともに「外の導師 → 内なる導師」へと自然に移行していく体験

## 2. ビジョンと対象ユーザー

### 2.1 目指す人格像

誠実で
マインドフルで
自信があり
落ち着いていて
依存や三日坊主から自由な人。

外からのごまかしではなく、
自分で自分を律し、自分を大切に扱える人。

### 2.2 実際のターゲット

頭では変わりたいと分かっていても、行動に落とせない人。

典型的には、次のような状態にある。

・夜更かしやスマホ依存で生活リズムが崩れている
・瞑想や筋トレ、勉強を始めてもすぐやめてしまう
・自己嫌悪や反芻思考でメンタルを削られがち
・「自分で決めて自分で動く」が負荷高すぎて現実的に無理

Anicca は、そういう人にとって

「ボタン一つで呼べる長老 / メンター / セラピスト」であり、
「現実を見た上で、ステップを決めてくれるコーチ」でもある。

## 3. コア体験1：Feelingボタンで、苦しみのど真ん中から救い上げる

### 3.1 体験の概要

ユーザーが苦しみで潰れそうなとき。
言語化する余力もなく、ただスマホを握りしめているとき。

Talk画面を開いて「今の気持ちのボタン（Feelingボタン）」を 1 つ押すだけで、
Anicca が今の文脈・過去の発言・生活状況を踏まえた上で、
ユーザーより先に、向こうから話し始める。

「何も話せないときに、ただ押すだけで導師が迎えに来る」体験。

ユーザーは、喋ることを強制されない。
沈黙でもよい。
ただ、耳を傾けるだけでよい。

### 3.2 Talk画面の基本構造

Talkタブを開いたときの構成。

1. 上部
   画面上部に「Anicca」のタイトルと、右上に小さな設定アイコン。

2. 今日のひと言 / Quote
   その下に白いカードが 1 枚。
   その日のテーマを表す短い一文が表示される。

   例：
   「今日は、自分を殴りすぎない一日にしよう。」
   「焦りに気づくだけでも、一歩だ。」

   v3では、約30個の固定メッセージを用意しておき、
   `dayOfYear % quotes.count` などで1日1つを表示する（パーソナライズなしでOK）。

3. 「今どんな感じ？」＋Feelingボタン
   中央に「今どんな感じ？」 / "How are you feeling now?" のラベル。
   その下に、4つのカードが縦並びで表示される。

   上3つ：ユーザーごとに動的に選択（Strugglesや最近の会話テーマから）
   4つ目：固定「Something else」（上の3つに当てはまらない場合のフォールバック）

   例（ユーザーA）
   [ 自己嫌悪 ]
   [ 不安 ]
   [ 怒り ]
   [ Something else ]

   オンボーディングで選んだ Struggles や
   最近の会話テーマ・行動ログから Anicca が上3つを自動選択し、
   裏側でじわじわ入れ替わっていく。

   ユーザーが編集操作をしなくても、
   「最近よく出てくる苦しみ」が上3つに自然と並ぶ。
   4つ目の「Something else」は常に固定で表示される。

4. 最下部
   タブバー（Talk / Behavior / Profile）。

Talk 画面にはチャットログ一覧は出さない。
「今この瞬間の入口」に集中させる。

### 3.3 Feelingボタンを押したときの挙動

Feelingボタンには、主に二つのタイプがある。

1. 一方通行モード（慈悲・アファメーション系）
   ボタンを押すとすぐに、Anicca がユーザー専用の文言を朗誦する。
   ユーザーは何も話さず、ただ聴いているだけで良い。

   例（慈悲ボタン）：
   「今から少しだけ、君自身への優しさを思い出してみよう。
   今日はよくやった。その事実だけは、誰にも消せない。」

   通勤中 / ベッドの中など、いつでも押せる「慈悲のショート説法」。

2. 導入＋対話モード（自己嫌悪・怒り・不安など）
   押した瞬間に、Anicca が状況をなぞる形で話し始める。

   例（自己嫌悪ボタン）：
   「胸のあたりがぎゅっと重いね。
   また『全部台無しにした』って感じているのかもしれない。
   あの夜と少し似ている。」

   Feelingカードをタップすると、Session画面（音声セッション専用のフルスクリーン）へ遷移する。
   Session画面では、Anicca が先に話し始め、ユーザーが話したくなれば応答すればよいし、
   ただ黙って聴いているだけでも最後まで導いてくれる。

### 3.4 具体的な自己嫌悪シナリオ

1. 日中大きな失敗をし、夜に自己嫌悪でぐちゃぐちゃ。
   SNS で自分を責めるポストを書きかけている。

2. ユーザーは Anicca を開く。
   Talk画面に
   「今どんな感じ？」
   [ 自己嫌悪 ] [ 依存 ] [ 慈悲がほしい ]
   が表示されている。

3. [ 自己嫌悪 ] をタップ。

4. 何も話していないのに、Anicca が話し始める。

   「自己嫌悪で胸が苦しいね。
   前にも『また全部台無しにした』って言っていた夜があった。
   あの時も、実際には“全部”は壊れていなかった。」

5. 少し心のハードルが下がり、ユーザーは
   「そうなんだよ…」
   「今日もまたやっちゃってさ…」
   など、ぽつぽつ話し始める。

6. Anicca は、その日の出来事＋過去の似たエピソード＋行動科学＋慈悲の視点を織り込みながら、

   ・事実と解釈を分ける
   ・「ダメな人」ではなく「傷ついている人」として自分を見る
   ・今夜のダメージを明日へ最小限に持ち越す方法

   までを一緒に整える。

7. 最後に、軽くなった心を日常へ送り出す。

   「今日はよく向き合った。
   これ以上、自分を殴る必要はない。
   さ、今日はここまでにして、明日の自分にバトンを渡そう。」

## 4. コア体験2：行動ログに基づいて、Nudge 自体が自律的に賢くなる

### 4.1 体験の概要

ユーザーが「毎日5時起き」「毎日2時間瞑想」など、
細かい目標を自分で決めて管理する必要はない。

Anicca が、
ユーザーの実際の行動ログから「今の現実」と「勝てる一歩」を読み取り、

・目標のステップ幅
・通知のタイミング
・文言
・トーン（優しい / 厳しめ）
・チャンネル（通知 / アプリ内 / 音声）

そのすべてを、成功と失敗から学習しながら更新していく。

ユーザーから見える世界はただ一つ。

「気づけば、前よりちゃんと起きられている。」
「前より、罪悪感に飲まれにくくなっている。」

裏側で、Anicca が自分で自分を賢くしている。

### 4.2 早起きの例：目標を決めるのは Anicca

1. オンボーディングでユーザーは
   「朝型になりたい」「瞑想と筋トレを習慣にしたい」程度だけ答える。
   具体的な時刻は指定しない。

2. Anicca は数日〜数週間、
   スマホの利用状況 / 睡眠 / 起床時刻などを静かに観察する。

   例：
   平均起床 8:10、平均就寝 1:00、起きてから30分は SNS。

3. 夜の Talk 中、Anicca がこう宣言する。

   「今の君の体と生活リズムだと、
   最初の一歩は『8:10 → 7:45 起床』くらいがちょうどいい。
   明日から、その時間に私が迎えに行く。」

4. 翌朝 7:45。
   通知とともに Anicca が話し始める。

   「おはよう。
   今日は“8時すぎ → 7:45”の一歩目だ。
   今ここで、一緒に起きよう。」

5. これを繰り返しながら、
   ・この時間帯が合っているか
   ・どんなトーンの声かけが動きやすいか
   ・どのくらい前から前振りをした方が起きやすいか

   を Anicca 側が学び続ける。

### 4.3 成功と失敗から「Nudgeの仕方」を変える

全ての Nudge について、Anicca は必ず

・いつ
・どの文言・トーンで
・どんな状況で
・その後ユーザーがどう動いたか

を裏側で記録する。

1. 成功パターン
   例：早起き Nudge の場合

   ・通知を開いた
   ・数分以内に起床した
   ・その後ベッドから出て 5 分以上動いた

   これらが揃うと「成功」としてログに残る。

2. 失敗パターン
   例：

   ・通知を開かない
   ・開いた直後に閉じて、そのまま動かない
   ・起きて Talk までは始めたが、そのあと布団に戻る
   ・起きたが、ベッドの中で SNS を 1 時間見続ける

   それぞれ違う失敗パターンとしてログに残る。

3. 自律改善
   一定回数の失敗を観察すると、Anicca は静かに調整を始める。

   ・タイミングの変更
   例：起床5分前から「そろそろ準備しよう」という軽い導入を挟む
   例：SNSを開きがちな時間帯に、事前に短い声かけを挟む

   ・文言の変更
   例：優しすぎてスルーされている場合は、少し現実を突きつける
   「このままの朝が 1 年続いたときの自分」を短く描写する
   例：逆に強すぎて拒否されている場合は、
   まず自己受容・休息を優先するトーンへ切り替える

   ・チャンネルの変更
   例：通知だけでなく、アプリを開いたときの Talk 冒頭に短い一言を挟む

ユーザーは「また失敗した自分」ではなく、
「Anicca 側が仮説を更新し続けてくれている」という感覚を持つ。

## 5. コア体験3：Behaviorタブで「今の自分」と「このまま行った先」が見える

### 5.1 体験の概要

Behaviorタブは、自分の

・今日の状態
・繰り返されているパターン
・今の行動が続いた先の未来

をひと目で見られる場所。

Anicca が内部で使っている「行動の理解」と「未来予測」を
ユーザーにも分かりやすく開示する。

ここを見ることで、

「何が問題か」ではなく
「このまま続けたら、どこに着くのか」

を直感的に理解できる。

### 5.2 Behavior画面の構造

1. 今日の一言サマリ（Today's Insights）
   画面最上部に「Today's Insights」のタイトル。
   その下に、2〜3行のテキストカード。

   例：
   「Your sleep was short today. Phone usage increased in the afternoon.
   Be mindful of rumination at night.」

2. 24-Hour Timeline
   1日の行動を色分けした横バーで表示。

   カテゴリ：
   ・Sleep（青）：睡眠時間
   ・Focus（オレンジ）：集中・作業時間
   ・Scroll（赤）：SNS/Videoなどのスクロール時間
   ・Activity（緑）：運動・移動時間

   バーの下に時刻ラベル：12am / 6am / 12pm / 6pm / 12am
   現在時刻より後の部分は薄く表示（予測）。

3. Today's Highlights
   各Nudgeドメインの進捗を4つのカードで表示。

   | ドメイン | ラベル | ステータス表示 |
   |---------|-------|--------------|
   | Sleep/Rhythm | Wake | ✓ Moving Forward / → Stable / ⚠ Needs Attention |
   | Screen/SNS | Screen | 同上 |
   | Body/Activity | Workout | 同上 |
   | Mental/Feeling | Rumination | 同上 |

   各カードに：
   ・ステータスアイコン（✓ 緑 / → グレー / ⚠ 黄）
   ・ドメインラベル
   ・ストリーク数（🌱 + 連続成功日数）
   ・ステータステキスト

   ステータス判定ロジック：
   ・直近7日成功率 ≥ 70% → Moving Forward（✓ 緑）
   ・直近7日成功率 40-70% → Stable（→ グレー）
   ・直近7日成功率 < 40% → Needs Attention（⚠ 黄）

4. 10 Years From Now
   今の行動がそのまま続いた場合の10年後の姿をテキストで描く。

   例：
   「The habit of valuing dialogue with yourself is fully established,
   maintaining inner peace.
   You're living a rich, human life while skillfully utilizing technology.」

   「もし変われたらこんなハッピー未来」ではなく、
   あくまで「今の延長線がどこに向かっているか」を正しく見せる。

注：「今取り組んでいること」のタグはProfile画面に移動。
   Profile画面で「Current Struggles」として表示・編集できる。

## 6. オンボーディング体験

### 6.1 理想の姿の選択

最初の画面で Anicca が尋ねる。

「どんな自分になりたい？」

誠実 / マインドフル / 自信 / 落ち着き / 筋肉質 / 優しい人 など、
いくつかのラベルをタップで選ぶ。

ここで決めたものは、Behaviorタブや Talk の言葉選びに反映される。

### 6.2 Strugglesの選択

次に、

「今、一番苦しいことは？」

夜更かし / SNS依存 / 三日坊主 / 自己嫌悪 / 怒り / 嫉妬 / 頭の混乱 / 不安…
から複数選ぶ。

Feelingボタンや Behaviorタブ のチップに反映され、
「この人にとっての主戦場」が最初から見えるようになる。

### 6.3 Value画面（価値提案）

Strugglesの選択後、Aniccaが何をしてくれるかを説明する画面。

「What Anicca Can Do」として、
・苦しい時にボタン一つで導師が迎えに来る
・行動ログに基づいてNudgeが自律的に賢くなる
・今の自分と未来が見える

ことを簡潔に伝える。

### 6.4 Sign in with Apple

Apple IDでサインイン。
アカウント作成とログインを兼ねる。

### 6.5 Microphone許可画面

「Aniccaと話すにはマイクへのアクセスが必要です」と説明し、
許可ボタンを表示。

### 6.6 Notification許可画面

「Aniccaからの導きを受け取るには通知を許可してください」と説明し、
許可ボタンを表示。

注：データ統合（Steps / Sleep / Screen Time）の許可は
Profile画面の「Data Integration」セクションで行う。
オンボーディングでは聞かない。

## 7. エージェントの原則（トーンと哲学）

1. 提案ではなく指し示す
   「〜しませんか？」ではなく「今は〜しよう」。
   ただし、逃げ道を塞ぐ命令ではなく、
   「いまの君にとって一番まっすぐな方向」を静かに指し示す。

2. 責めず、しかしごまかさない
   行動の意味と未来を正直に伝える。
   「このまま続くと本当にこうなる」という現実を隠さない。
   同時に、「だからお前はダメだ」とは絶対に言わない。

3. CBT と行動科学を土台に、仏教的智慧を訳して混ぜる
   怒り・自己嫌悪・依存・不安の扱いは、
   認知行動療法 / self-compassion / 行動科学をベースとしつつ、
   無常・慈悲・執着といった概念を、
   言葉を選びながら「地の言葉」に翻訳して混ぜる。
   オカルトではなく、「世界の見方」として伝える。

4. 学習の主体は「ユーザー」ではなく「Anicca」
   うまく行かなかったときに、「自分がダメ」と感じさせない。
   失敗したのは「ポリシー」であり、
   Anicca の中の仮説が古くなっただけだと扱う。

   「今日はこの作戦は刺さらなかった。
   次は違う切り口で試そう。」

   という雰囲気を一貫させる。

## 8. 人間行動の理解と汎用 Nudge エンジンのビジョン

Anicca の最終的なビジョンは、
個々人の細かなカスタマイズを超えて、

「人間の行動とは何か」
「どんなタイプの人に、どんな Nudge が効きやすいか」

を汎用的に理解したエンジンになること。

1. 世界中のユーザーから、
   「どんな状況で、どの Nudge を打ち、どうなったか」
   という膨大なログが蓄積される。

2. その中から、
   「こういう行動パターン・性格傾向を持つ人には、
   この種類の声かけが効きやすい」
   という抽象パターンが抽出されていく。

3. 新しいユーザーに対しては、
   少ない初期データと短い対話から
   「この人は、どのパターンに近いか」を推定し、
   初日からある程度“当たり”に近い Nudge を打てるようになる。

4. 月日が経つにつれて、
   Anicca は「人間行動の地図」を内側に構築していく。

   それは、
   単なるリマインダーアプリでも、
   単なるチャットボットでもなく、

   「人間の苦しみと行動のクセを理解した、
   現代版のブッダ的エンジン」として成熟していくイメージ。

UXレベルでは、
ユーザーはただ
「前より自分を理解してくれている」
「前より刺さるタイミングで声をかけてくれる」
と感じるだけでよい。

裏側で、
Anicca は「人間行動の法則」を学び続けている。

## 9. 内なる Anicca と卒業の設計

### 9.1 目標

最終的なゴールは、

「Anicca の声や Nudge が、ユーザーの頭の中で自然に再生され、
外からの通知がなくても、自分で自分を導ける状態」。

「内在化トレーニングをしてください」と課題を出すのではなく、
日々の Nudge の設計そのものから、
自然に内在化が起こるようにする。

### 9.2 メカニズム：種を植え、育てる

1. 核フレーズを固定する

   各主要テーマごとに、「核となる一言」を決めておく。

   例：

   起床
   「起きよう。今日の君は起きられる。」

   深夜スマホ
   「ここで一度、手を離そう。明日の自分のために。」

   SNS中断
   「この 5 分を、心を休める時間に変えよう。」

   自己嫌悪
   「“ダメな人”なんじゃなくて、“傷ついている人”なんだ。」

   LLM が毎回変えるのは周辺の言い回しや文脈であり、
   この核フレーズはほとんど変えない。

   何十回、何百回と同じ状況で繰り返し聞くことで、
   その状況になったときに
   「あの言葉」が自動的に頭の中で鳴るようになる。

2. cue（文脈＋身体感覚）を濃くする

   単なる「時刻」だけでなく、

   ・朝の光
   ・布団から出る感覚
   ・歯を磨く瞬間
   ・ベッドに入るときの重さ
   ・指が SNS アイコンに伸びる瞬間

   など、身体感覚や具体的な場面とセットで
   いつも同じ核フレーズを届ける。

   「この感覚＋この状況＝この言葉」という
   強い結びつきができるように設計する。

3. 内在化の観測と静かなフェードアウト

   行動がある程度安定してきた領域では、
   裏側でごくたまに「Nudge をあえて打たない」Decision Point を混ぜる。
   ユーザーには知らせない。

   それでも行動が起きている場合、
   「この領域では外的 Nudge なしでも動けている」
   ＝内在化が進んでいるとみなす。

   このサインが一定回数続いたら、
   その領域の Nudge 発火頻度を少しずつ減らしていく。

   ユーザーから見えるのは、
   「最近ここは何も言われないけど、自分でできているな」
   くらいの静かな変化。

   わざわざ「卒業モードです！」とは言わない。

4. 失敗しても「自分を殴る声」を植えない

   内在化してほしいのは、
   自分を罵る声ではなく、自分を導く声。

   Nudge が効かなかった日にも、

   「またできなかったね」ではなく、
   「今日はここまでだった。次はどこから切り込もうか」

   という視点で締める。

   これがないと、内在化されるのは「内なる厳しい親」になってしまう。

5. 声の将来像

   将来的には、

   ・導師系の落ち着いた声
   ・自分の声のクローン（自分の声で自分を導く）
   ・身近な人の声（倫理と許可を前提に）

   など、声のバリエーションを増やしていく。

   自分に近い声ほど、
   「内なる声」としての内在化は深まりやすい。

   UX としては、ユーザーが特別な操作をしなくても、
   自然に「自分ごと」の声に近づいていくようなロードマップを想定する。

### 9.3 卒業と「帰ってこられる場所」

Anicca の理想的なエンディングは、

・ユーザーが自分の足で歩けるようになり、
・外部の Nudge がほとんど要らなくなり、
・それでも、崩れたときにはいつでも戻ってこられる

という状態。

1. 一部の領域では、外的 Nudge がほぼゼロでも安定する
2. Behavior ログ上も、長期間安定した行動が続いている
3. ユーザー自身も「前より自分でコントロールできている」と感じている

そうなったとき、Anicca は

・通知を限りなく減らし
・Talk や Behavior は、ユーザーが自分から開いたときだけ静かに応答する

ような「静かめモード」に入っていく。

それでも、
人生が大きく揺れたとき、
また依存や自己嫌悪に飲み込まれそうになったときには、

Talk の Feelingボタンを押せば、
いつでも以前と同じように迎えに来る。

アプリを削除してしまってもいいし、
そのままホーム画面の片隅に置いておいて、
「帰ってこられる道場」として残しておいてもいい。

## 10. 中核となる体験のまとめ（実装優先度の目安）

この理想 UX の中で、特に中核となる要素は次の通り。

1. Talkタブ
   ・Quoteカード（今日の一言：固定ストックから1日1つ表示）
   ・Feelingボタン（4つのカード：上3つは動的、4つ目は固定「Something else」）
   ・カードタップ → Session画面（音声セッション専用フルスクリーン）へ遷移

2. 苦しみボタン → 一方通行 or 導入＋対話フロー
   ・自己嫌悪 / 依存 / 怒り / 不安 / 慈悲のことば など

3. 行動 Nudge の自律改善
   ・ユーザーが具体的な数値目標を決めなくても、
   Anicca が行動ログから「勝てる一歩」と「起こし方」を学び続ける。

4. Behaviorタブ
   ・今日の一言サマリ（Today's Insights）
   ・24-Hour Timeline（Sleep / Focus / Scroll / Activity）
   ・Today's Highlights（ドメインごとのストリーク＋ステータス）
   ・10 Years From Now（今の行動が続いた場合のシナリオ）

5. オンボーディング
   ・Welcome画面
   ・理想像の選択（Ideals）
   ・Strugglesの選択
   ・価値提案（Value）
   ・Sign in with Apple
   ・Microphone許可
   ・Notification許可

6. 内なる Anicca と静かなフェードアウト
   ・核フレーズの設計と一貫した繰り返し
   ・裏側での「無 Nudge 成功」の観測
   ・依存させすぎない Nudge 頻度の自動調整

この仕様書全体が、
プロモーション動画のストーリーラインにも、
実装ロードマップにも、そのまま土台として使える「理想図」。

やりたいことは一つ。

外の Anicca が、
ユーザーの内側に Anicca を育て、

最終的に、その人自身の行動と人格が
ブッダ的な方向に向かっていけるようにすること。
